# VeilSwap

> Шаблон репозитория для смарт‑контрактов и инфраструктуры VeilSwap (Hardhat + Solidity).

⚠️ **Важно про “полную анонимность без авторизации”.** Я не могу помогать с реализацией обменника, который
специально предназначен для сокрытия происхождения средств/обхода комплаенса или облегчения отмывания денег.
Но я могу помочь сделать **инженерный каркас репозитория** и подсказать, как корректно интегрировать ZK для
*privacy-by-design* (например, селективное раскрытие, доказательство свойств без раскрытия данных) и как
спроектировать безопасные контракты и тесты.

## Структура

- `contracts/` — Solidity контракты (пример: минимальный Router + интерфейс верификатора ZK).
- `scripts/` — деплой/утилиты Hardhat.
- `test/` — тесты (Mocha/Chai).
- `circuits/` — место под ZK‑схемы (положен “hello world” пример‑заглушка).
- `.github/workflows/` — CI (lint + test).

## Быстрый старт

```bash
cp .env.example .env
npm i
npm run compile
npm test
npm run deploy:local
```

## ZK интеграция (безопасный минимум)

В `contracts/interfaces/IZKVerifier.sol` — интерфейс верификатора, который можно подключить к реально
сгенерированному verifier-контракту (Groth16/PLONK и т.п.). В этом шаблоне **нет** “анонимайзера” или “миксера” —
только точка интеграции.
